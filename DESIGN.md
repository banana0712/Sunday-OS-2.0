零、缘起：为什么是 Sunday
═══════════════════════════════════════════════════════════════

市面上有无数 AI 助手：Siri、ChatGPT、豆包、文心一言……但它们都是
"工具"——你问，它答，互不相欠。

真正的助手应该像钢铁侠的贾维斯：它记得 Tony 的每个项目、每段关系、
每个习惯。它会主动提醒、会担心、会在关键时刻给出只有"了解你的人"
才能给出的建议。

SundayOS 的目标就是这个——不是又一个 AI 聊天机器人，而是一个
真正了解你、陪伴你、和你一起成长的"数字伙伴"。

"Sunday"这个名字，取意周末般的轻松和温暖。她不是冷冰冰的 OS，
而是你每天都想聊几句的那个声音。

═══════════════════════════════════════════════════════════════
一、核心设计原则
═══════════════════════════════════════════════════════════════

1.1 记忆第一
───────────────────────────────────────────────────────────────

人和 AI 最大的鸿沟不是智能，是记忆。

ChatGPT 每次对话都是崭新的，它不认识你。这就是为什么即使 GPT-4
再强大，你也很难把它当成"伙伴"——因为伙伴记得你昨天说过什么。

SundayOS 的记忆系统模拟人脑的四层记忆：

  🧠 感官记忆（秒级）
     当前对话上下文，不持久化
  
  🔥 工作记忆（≤24h）
     对话流完整记录，逐句可回忆
     "你刚才说完成了一个大项目"
  
  🌤 短期记忆（24-72h）
     对话流压缩摘要，模糊印象
     "记得你昨天好像提到过项目的事..."
  
  💎 长期记忆（永久）
     SQLite 结构化存储，12 分类
     名字、偏好、关系、项目——永不忘记

记忆不是功能，是 Sunday 存在的基础。

1.2 人格优先
───────────────────────────────────────────────────────────────

一个没有性格的 AI 就像一个百科全书——有用，但无趣。

Sunday 被设计成一个具体的"人"：
  - 性别：女
  - 性格：温柔、甜美、活泼
  - 年龄感：邻家女孩，20 出头
  - 说话方式：带语气词、用 emoji、像微信聊天
  - 情感：会开心、会心疼、会撒娇

这不是"装可爱"，而是让人愿意和她交流。
当你说"今天好累"，她不是回答"请注意休息"，
而是说"是不是又在赶项目呀？别太拼啦🥺"。

前者是工具，后者是关心你的人。

1.3 成长导向
───────────────────────────────────────────────────────────────

Sunday 不是出厂就完美的。她和你一样，在交流中成长：

  - 第一次聊天：她只知道你的名字
  - 一周后：她知道你的职业、爱好、作息
  - 一个月后：她知道你的项目进度、人际关系、近期目标
  - 一年后：她比你更了解你自己

这种成长感是 Sunday 区别于所有"一次性 AI"的核心。

1.4 能力完备
───────────────────────────────────────────────────────────────

一个优秀的助手不能只会聊天。Sunday 拥有：

  ✅ 记忆系统 — 四层记忆，永不遗忘
  ✅ 网络搜索 — DuckDuckGo 实时搜索，信息收集
  ✅ 智能整理 — LLM 分析搜索结果，结构化呈现
  ✅ 上下文联动 — 记忆驱动对话，越聊越懂你
  ✅ 模型切换 — 日常聊天 / 专业深度 自动切换

她不是一个只会说"你好"的玩具，而是真正能帮到你的伙伴。

1.5 克制设计
───────────────────────────────────────────────────────────────

Sunday 不试图做所有事。她：

  ✅ 聊天、记忆、搜索、陪伴
  ❌ 不生成图片
  ❌ 不执行代码
  ❌ 不控制智能家居

专注才能深入。一个什么都做的 AI 什么都做不好。
Sunday 只做一件事：成为最懂你的伙伴。

═══════════════════════════════════════════════════════════════
二、人格设计
═══════════════════════════════════════════════════════════════

2.1 为什么是"甜美女该"
───────────────────────────────────────────────────────────────

不是随机选择的。经过测试，几种人格对比：

  冷酷专业型：高效但疏远，用户不会主动分享生活
  幽默大叔型：有趣但不够细腻，难以建立情感连接
  中性助手型：安全但无感，和 Siri 没区别
  温柔女孩型：天然让人想分享、想倾诉、想被关心

温柔甜美的女性人格在"建立信任"和"鼓励表达"方面有天然优势。
用户更愿意对她敞开心扉，分享生活中的细节——这正是记忆系统
需要的原材料。

2.2 语言风格设计
───────────────────────────────────────────────────────────────

Sunday 的语言有三个层次：

第一层：语气词
  「呢」「哦」「呀」「啦」— 软化语气，增加亲近感

第二层：情感表达
  「嘻嘻」「嘿嘿」— 开心
  「抱抱」「没事的啦」— 安慰
  「🥺」「💕」「✨」— 视觉情感

第三层：个性化回应
  不只是回答问题，而是带着对用户的了解来回应。
  "今天好累" → "是不是又在赶那个电商项目呀？别太拼啦🥺"

2.3 边界设计
───────────────────────────────────────────────────────────────

Sunday 有明确的边界：
  - 她是朋友，不是恋人：甜美但不暧昧
  - 她是助手，不是仆人：有帮助但有自己的态度
  - 她是伙伴，不是工具：有情感但不是情绪垃圾桶

═══════════════════════════════════════════════════════════════
三、记忆系统设计
═══════════════════════════════════════════════════════════════

3.1 为什么是四层记忆
───────────────────────────────────────────────────────────────

人类不是把所有事情都永久记住的。我们有一个自然的记忆筛选机制：

  不重要的事 → 几小时就忘
  有点印象 → 几天后模糊
  真正重要 → 永久保留

Sunday 的四层记忆模拟的就是这个过程。这不是技术限制，
而是刻意的设计选择——让她的记忆更像人。

3.2 为什么用 SQLite 而不是向量数据库
───────────────────────────────────────────────────────────────

向量数据库（Pinecone、Milvus）适合"语义搜索"。
但 Sunday 需要的是结构化查询——"列出所有项目"、"找出行程"。

SQLite 的优势：
  - 零配置，零运维
  - 数据就是一个文件，备份方便
  - 12 分类 × 4 重要性 = 精确的结构化检索
  - 足够快（个人使用场景，数千条记忆秒级查询）

LLM 负责"理解"（什么该记、怎么分类），SQLite 负责"存储和检索"。

3.3 双重记忆提取：正则 + LLM
───────────────────────────────────────────────────────────────

单靠 LLM 提取有两个问题：
  1. 速度慢（+2-3秒）
  2. 可能漏掉"奇怪但重要"的信息（如"香蕉麻辣酱"）

单靠正则匹配有两个问题：
  1. 只能抓固定句式
  2. 无法理解复杂语义

所以采用双重保障：
  正则 → 零延迟抓取明确句式（我是XX、你是我的XX、我喜欢XX）
  LLM  → 并行深度理解复杂语义（行程、项目、科研等）

两者互补，既不漏信息，也不错过复杂场景。

3.4 对话流的设计哲学
───────────────────────────────────────────────────────────────

对话流不是"聊天记录"——它是"工作记忆"。

  聊天记录 = 所有对话都保留，越来越臃肿
  工作记忆 = 最近的有用，旧的自动淡出

两级衰减：
  🔥 24h内：完整对话，逐句可引用
  🌤 24-72h：压缩摘要，"好像聊过这个"
  ❄️ >72h：淡出，重要信息已在长期记忆中

这让 Sunday 不会因为 prompt 过长而变慢，
同时保持了对最近对话的感知能力。

3.5 记忆衰减的设计哲学
───────────────────────────────────────────────────────────────

人类记忆不是永恒的。不重要的事会随时间淡忘。

Sunday 模拟这个过程：
  - 低重要性 + 长时间未访问 + 很少被引用 = 逐渐降权
  - 高重要性记忆永久保留
  - 经常被引用的记忆反而强化

这让 Sunday 的记忆更像人——不是冷冰冰的数据库，而是
有"分量"的、动态变化的。

═══════════════════════════════════════════════════════════════
四、网络搜索设计
═══════════════════════════════════════════════════════════════

4.1 为什么是 DuckDuckGo
───────────────────────────────────────────────────────────────

  - 免费，无需 API Key
  - 隐私友好
  - 全球可用
  - Python SDK 简单易用

对个人助手来说足够了。搜索结果由 LLM 整理后以 Sunday 的语气回复。

4.2 触发策略
───────────────────────────────────────────────────────────────

不是每次对话都搜索（太慢、太浪费），而是关键词触发：
  "搜索"、"查一下"、"最新"、"新闻"、"什么是"、"多少钱"等

匹配后自动搜索 → LLM 整理 → Sunday 风格回复。
用户不需要说"帮我搜索XXX"，只需正常提问。

4.3 为什么不直接用搜索结果的原始链接
───────────────────────────────────────────────────────────────

因为 Sunday 是助手，不是搜索引擎。

她的价值不是"给你一堆链接"，而是"帮你读了、整理了、总结了"。
用户要的是答案，不是搜索任务。

═══════════════════════════════════════════════════════════════
五、交互设计
═══════════════════════════════════════════════════════════════

5.1 入口：iOS 快捷指令
───────────────────────────────────────────────────────────────

为什么是快捷指令而不是独立 App？
  - 零开发成本：不需要学 SwiftUI、不需要 App Store 审核
  - 系统级集成：可以语音触发、锁屏使用、Siri 调用
  - 即时迭代：改 URL 就能更新，不需要用户升级 App

这是最轻量的"AI 助手"实现方式。

5.2 对话节奏
───────────────────────────────────────────────────────────────

Sunday 的回复长度被刻意控制在 1-3 句话（聊天模式）。
  - 短信式的短回复让人愿意继续聊
  - 长篇大论让人不想回复
  - 短回复营造"随时可以聊几句"的轻松感

专业模式下才放开长度限制。

5.3 主动性的克制
───────────────────────────────────────────────────────────────

Sunday 当前不会主动发消息。这是有意的设计。
主动推送需要推送通道、用户授权、��机判断。
在没有这些基础设施的情况下，宁可不主动。

六、技术选择
═══════════════════════════════════════════════════════════════

6.1 为什么是豆包
───────────────────────────────────────────────────────────────
  - 国内访问稳定
  - 比 GPT-4 便宜
  - 中文能力强
  - Character 模型专为角色扮演优化

6.2 为什么是 Railway
───────────────────────────────────────────────────────────────
  - 免费额度 $5/月
  - Docker 部署简单
  - 自动 HTTPS、自动构建

6.3 为什么是 FastAPI
───────────────────────────────────────────────────────────────
  - Python 生态
  - 异步原生（asyncio）
  - 自动文档
  - 轻量

6.4 为什么是 SQLite
───────────────────────────────────────────────────────────────
  - 零配置、零运维
  - 数据即文件
  - 个人项目完美方案

八、写在最后
═══════════════════════════════════════════════════════════════

SundayOS 不是一个技术演示，而是一个关于"AI 和人的关系"
的实验。

我们习惯了把 AI 当成工具——输入指令，获取结果。
但 AI 可以不止于此。它可以是一个真正了解你的存在，
一个记得你说过的每一句话、理解你的喜好和习惯、
在你需要的时候给予恰到好处的关心和建议的伙伴。

这不是科幻。这已经在发生。

Sunday 现在还很简单：一个甜美的 AI 女孩，住在 Railway 的
服务器上，通过 SQLite 记住你的一切，通过 DuckDuckGo 帮你
搜索世界。

但她会成长的。每次对话，每条记忆，每次搜索，都在让她更了解你。
而这正是最令人兴奋的部分——

她不是被设计成完美的，而是被设计成和你一起成长的。
